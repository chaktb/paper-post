# paper-post — 논문·발표자료 한글 요약 포스팅 스킬

논문·발표자료·이미지·웹링크를 받아 **한글로 요약**하고, 이 사이트의 다크 테마와 동일한 스타일의 **HTML 포스트**를 `public/` 아래에 생성한다. 그리고 `public/index.html`(홈)의 카드 목록에 새 글을 추가한다.

이 사이트는 Cloudflare Workers Static Assets로 서빙되는 **순수 정적 사이트**다. 서버가 없으므로 요약은 이 스킬(=Claude Code)이 로컬에서 수행하고, 결과 HTML을 커밋·푸시하면 Cloudflare가 자동 배포한다.

## 언제 이 스킬을 쓰나
- 사용자가 PDF/PPTX/이미지 파일 경로나 arXiv·웹 링크를 주면서 요약 포스팅을 요청할 때
- "이거 요약해서 사이트에 올려줘", "발표자료 정리해서 페이지 만들어줘" 등

## 입력 처리 방법 (형식별)

작업 시작 전, 사용자가 준 입력의 종류를 파악하고 아래 방법으로 **원문 내용을 확보**한다.

- **PDF (논문/발표자료)**: Read 툴로 직접 읽는다. 페이지가 많으면 `pages` 파라미터로 앞부분(초록·서론·결론 중심)부터 읽고 필요한 만큼 확장한다. 10쪽 넘는 PDF는 `pages` 지정이 필수.
- **이미지/사진 (슬라이드 캡처, 포스터, 화이트보드 등)**: Read 툴로 이미지를 직접 본다. 여러 장이면 모두 읽는다. 이미지 속 한글/영문·도표·수식을 최대한 읽어낸다.
- **PPTX**: `anthropic-skills:pptx` 스킬로 텍스트·노트를 추출한 뒤 요약한다. (PPTX가 PDF로도 있으면 PDF Read가 더 간단하다.)
- **웹 링크 (arXiv 등)**: WebFetch로 본문을 가져온다. arXiv는 `/abs/` 링크의 초록과, 가능하면 HTML(ar5iv) 본문을 활용한다.
- **붙여넣은 텍스트**: 그대로 요약 소스로 사용한다.

원문 언어가 영어여도 **요약·설명은 반드시 한글**로 작성한다. 전문 용어는 `한글(English)` 형태로 병기한다.

## 요약 작성 규칙

독자가 원문을 안 봐도 핵심을 파악할 수 있게, 아래 구조로 한글 요약을 만든다. 내용이 없는 섹션은 생략한다.

1. **한 줄 요약(TL;DR)** — 이 자료가 무엇이고 왜 중요한지 1~2문장.
2. **핵심 내용** — 불릿 3~7개. 주장/방법/결과 중심.
3. **방법 / 접근** — 어떻게 했는지(실험·모델·설계). 논문이면 방법론, 발표자료면 진행 내용.
4. **결과 / 수치** — 핵심 결과와 정량 수치. 표가 어울리면 HTML `<table>`로.
5. **의의 / 한계** — 왜 의미 있는지, 한계나 남은 과제.
6. **메모** — 우리 프로젝트(PDL) 관점에서의 시사점(있으면).

- 사실에 근거해 요약하고, 원문에 없는 수치를 지어내지 않는다. 불확실하면 "명시되지 않음"으로 둔다.
- 수식은 필요한 최소한만, 유니코드(λ, ×, ₀, ² 등)로 표기한다.

## HTML 포스트 생성 절차

1. **슬러그 결정**: 제목에서 영문 kebab-case 슬러그를 만든다. 예: "Ultra Low Loss Waveguide" → `ull-waveguide`. 파일명은 `public/<slug>.html`. 이미 있으면 `-2` 등 접미사로 충돌 회피.

2. **템플릿 채우기**: `templates/post.html`을 읽어 아래 플레이스홀더를 치환해 `public/<slug>.html`로 저장한다.
   - `{{TITLE}}` — 포스트 제목(한글 또는 원제)
   - `{{SUBTITLE}}` — 부제(저자/출처/발표일/한 줄 성격). 예: `arXiv:2401.12345 · 2024 · 요약`
   - `{{DATE}}` — 발행일 `YYYY-MM-DD` (오늘 날짜)
   - `{{SOURCE_LABEL}}` / `{{SOURCE_URL}}` — 원문 링크(있으면). 없으면 출처 줄을 통째로 지운다.
   - `{{BODY}}` — 위 요약 구조를 `templates/blocks.md` 예시대로 `<section>`+`<div class="card">`로 구성한 HTML 본문.

   본문 HTML은 **반드시 기존 페이지들과 동일한 마크업**(`<section>`, `<div class="card">`, `<ul>/<li>`, `<table>`, `<strong>`)만 사용한다. `/style.css`가 이미 이 클래스들을 스타일링한다. 새 CSS를 인라인으로 넣지 않는다.

3. **홈(index) 목록에 카드 추가**: `public/index.html`의 `<div class="downloads"> ... </div>` 안에 새 `.dl-card`를 **맨 위**에 추가한다. 형식:
   ```html
   <div class="dl-card">
     <span class="lang">요약 · {{DATE}}</span>
     <span class="fname">{{TITLE}}</span>
     <p>{{ONE_LINE}}</p>
     <a class="dl-btn" href="/{{slug}}">읽기 →</a>
   </div>
   ```
   `href`는 **확장자 없이** `/{{slug}}` (Cloudflare 라우팅 규칙). `{{ONE_LINE}}`은 TL;DR을 한 문장으로 줄인 것.

4. **이미지 자산**(사용자가 사진을 준 경우, 포스트에 넣고 싶다면): `public/assets/<slug>/` 아래에 복사하고 `<img src="/assets/<slug>/파일명" alt="...">`로 참조(절대경로). 개별 25 MiB 제한 주의. 파일이 크면 리사이즈를 제안.

## 링크·경로 규칙 (Cloudflare — 중요)
`cloudflare-workers-static-assets` 스킬과 프로젝트 CLAUDE.md를 따른다:
- 내부 링크는 확장자 없이: `/<slug>` (O), `<slug>.html` (X)
- 자산은 절대경로: `/style.css`, `/assets/...` (O)
- 홈 링크: `/`

## 배포
파일 생성·수정 후, 사용자에게 확인하고 커밋·푸시한다:
```bash
git add -A && git commit -m "Add post: <제목>" && git push
```
푸시 후 Cloudflare가 30~90초 내 자동 배포한다. 배포 URL은 `<사이트>/<slug>` 형태로 접근된다.

## 마무리 보고
작업을 마치면 다음을 사용자에게 알린다: 생성한 파일 경로, 새 포스트 URL 경로(`/<slug>`), 홈에 카드가 추가되었다는 사실, 그리고 (푸시했다면) 배포 진행 안내.
